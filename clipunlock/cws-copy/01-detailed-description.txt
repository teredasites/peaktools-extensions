WHY COPYUNLOCK?

Unlike simple copy unlockers that only remove CSS user-select, CopyUnlock detects restrictions across 5 categories:

1. CSS blocks — user-select:none, pointer-events:none, -webkit-touch-callout, ::selection hiding
2. JavaScript handlers — oncopy, onpaste, oncut, oncontextmenu, onselectstart, onkeydown, alert() traps
3. DOM manipulation — MutationObserver re-blockers, overlay divs, inert attributes, Shadow DOM
4. Clipboard API interception — pastejacking, clipboardData.setData hijacking, getSelection override
5. Server-side protections — watermark injection, homoglyph substitution, custom font ciphers, image-based text, SVG path rendering, debugger traps, print blocking

Most copy protection uses 3-5 of these simultaneously. Other extensions only handle 1-2 categories and fail on heavily protected pages. CopyUnlock handles all 39 methods across all 5 categories.
